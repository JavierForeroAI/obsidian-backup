✅ Fortalezas

  • ✓ Teal diferencia (vs competencia azul profundo)
  • ✓ Futura Bold jerarquía clara
  • ✓ Espaciado respira bien (36px sections)
  • ✓ Copy honesto (reduce ansiedad)
  • ✓ Credibilidad visible (números, Dr. Carreño)

❌ Áreas de Mejora

  • ❌ Coral #F4A4A4 muy suave para CTAs
  • ❌ Body text Helvetica Light (fatiga ocular)
  • ❌ Contraste 44% apenas pasa WCAG AA
  • ❌ Credibilidad al final (debería ser arriba)
  • ❌ 3x CTAs repetidas = fatiga decisoria

Métricas Neurociencia Clave

Métrica Benchmark Interpretación
Fixation Duration <300ms scanned | 300-500ms normal | >800ms problema Tiempo que ojo se detiene. >800ms = no entiende (carga cognitiva alta)
Time to First Fixation (TTFF) <400ms alerta crítica (bueno) Cuánto tarda el ojo en notar un botón. Rojo <300ms. Coral actual >600ms.
Dwell Time Información crítica >3s Tiempo total en sección. Si <2s, probablemente saltó (sin procesar)
Pupil Dilation +0.8mm = carga alta Aumento pupilar = esfuerzo cognitivo. Teal reduce. Contraste bajo aumenta.
Scroll Depth >85% es objetivo ¿Qué % de usuarios leen hasta footer/CTA final?
Insight clave: Pacientes médicos ansiosos leer 3-4 minutos sin confianza visible → abandonan. Credibilidad (6.000 pacientes, 30+ profesionales) está en sección 4-5, pero deberían estar en HERO o sección 2.
Recomendaciones

Plan de Optimización (Priorizadas)

🔥 TIER 1 — HOY (15 min, impacto +25-35%)

R001: Botones Coral #F4A4A4 → #E8634A

Por qué: Coral desaturado = color de deliberación (lento). Rojo-naranja saturado = urgencia (amígdala activa). Rojo detectado en <300ms. Coral tarda >600ms.

CSS: background: linear-gradient(135deg, #E8634A 0%, #D1523D 100%); box-shadow: 0 6px 20px rgba(232, 99, 74, 0.35);

Impacto: +12-18% click-through rate

R002: Body Text Helvetica Light → Regular + Line-height 1.65

Por qué: Light-weight (300) causa flicker fatiga. Contraste 44% vs gris apenas pasa WCAG AA. Pacientes ansiosos necesitan velocidad de lectura, no fricción.

CSS: font-weight: 400; line-height: 1.65; color: #222;

Impacto: +8-15% scroll-depth, −30% abandono por cansancio

R003: Divisoras Teal Subtiles Entre Secciones

Por qué: Líneas guían fixations naturales (principio visual closure). Reducen carga cognitiva de parsing estructura. Teal = confianza (no alarma).

CSS: section { border-bottom: 1px solid rgba(32, 178, 170, 0.15); margin-bottom: 2.5rem; }

Impacto: +5-10% lectura de secciones completas

⏱️ TIER 2 — IMPORTANTE (1-2 semanas, impacto +10-15%)

R004: Tipografía Híbrida

h1 Futura (moderno) + h2-h3 Georgia serif (credibilidad médica). Contraste tipográfico = diferenciación visual + memoria más fuerte.

R005: 3D Buttons

Gradient + shadow profundo crea depth cue → botón percibido como "clickeable". Corteza parietal (affordance detection).

R006: Footer Stripe

Teal + white stripe genera contraste final → anclaje visual + memorización (serial position effect).

📊 TIER 3 — VALIDACIÓN (3-4 semanas, confirmación)

R007: A/B Test Hotjar

Split 50/50: botones coral actual vs #E8634A. 2 semanas, N=500 por rama. Medir: click-rate, scroll-depth, session-duration. Validación estadística (p<0.05).

R008: Eye-Tracking UserTesting

10 usuarios. Grabar fixation patterns, pupil dilation, blink-rate en hero + primeras 3 secciones. Validar si cambios tipográficos mejoran reading fluency. Costo: $150 one-time.

R009: Baseline Cognitivo

Medir HOY: tiempo lectura promedio, scroll-depth, drop-off por sección. Establecer baseline para medir mejora post-implementación.
